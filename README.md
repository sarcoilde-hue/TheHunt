# The Hunt - 非对称对抗游戏插件

一个类似于《黎明杀机》或《第五人格》的非对称对抗游戏插件，适用于 Minecraft 服务器。

## 游戏概述

- **杀手**：1-2名玩家扮演杀手，目标是在限定时间内击杀所有求生者
- **求生者**：4-8名玩家扮演求生者，目标是在限定时间内生存或击杀所有杀手
- **游戏时间**：默认10分钟（可配置）
- **动态角色分配**：根据玩家人数自动调整杀手数量

## 功能特性

### 核心功能
1. **等待大厅系统** - 玩家加入后在大厅等待，达到最小人数后开始倒计时
2. **随机出生点** - 杀手和求生者在预设的多个出生点中随机出生
3. **资源生成系统** - 地图中随机生成武器、工具、药水、食物和特殊道具
4. **自定义NBT道具** - 支持带有特殊属性的自定义物品
5. **倒计时系统** - 游戏时间倒计时，可配置
6. **轮廓显示系统** - 每2分钟显示10秒杀手轮廓（可配置）
7. **死亡观战系统** - 死亡后可以观战其他玩家
8. **游戏限制** - 禁止饥饿、自然回血、破坏方块和使用指令

### 游戏机制
- **杀手优势**：更多生命值、夜视效果、特殊武器
- **求生者策略**：收集资源、躲避杀手、团队合作
- **轮廓显示**：定期显示杀手位置，增加游戏紧张感
- **动态平衡**：根据玩家人数自动调整杀手数量

## 安装与使用

### 安装步骤
1. 将编译后的插件放入服务器的 `plugins/` 目录
2. 重启服务器
3. 插件会自动生成配置文件

### 基本命令

#### 玩家命令
- `/hunt join` - 加入游戏大厅
- `/hunt leave` - 离开游戏
- `/hunt info` - 查看游戏信息
- `/hunt help` - 显示帮助

#### 管理员命令
- `/huntadmin start` - 开启游戏大厅
- `/huntadmin stop` - 强制停止游戏（待实现）
- `/huntadmin reload` - 重载配置文件
- `/huntadmin setspawn <类型>` - 设置出生点（killer/survivor/resource）
- `/huntadmin status` - 查看游戏状态
- `/huntadmin help` - 显示管理员帮助

### 权限节点
- `thehunt.use` - 使用插件基础功能
- `thehunt.admin` - 管理员权限
- `thehunt.join` - 加入游戏
- `thehunt.leave` - 离开游戏
- `thehunt.start` - 开始游戏
- `thehunt.stop` - 停止游戏
- `thehunt.setup` - 设置游戏区域

## 配置说明

### 主要配置项

#### 游戏设置
```yaml
game:
  time: 600  # 游戏时间（秒）
```

#### 轮廓显示
```yaml
outline:
  interval: 120  # 显示间隔（秒）
  duration: 10   # 显示持续时间（秒）
```

#### 玩家设置
```yaml
players:
  min: 4            # 最小玩家数
  max: 10           # 最大玩家数
  killer-threshold: 8  # 超过此人数时使用2名杀手
```

#### 出生点设置
出生点使用格式：`世界名,x,y,z,yaw,pitch`
```yaml
spawns:
  killer:
    - "world,10,64,10,180,0"
  survivor:
    - "world,20,64,20,0,0"
  resource:
    - "world,30,64,30"
```

### 设置出生点步骤
1. 站在想要设置的位置
2. 执行对应命令：
   - `/huntadmin setspawn killer` - 设置杀手出生点
   - `/huntadmin setspawn survivor` - 设置求生者出生点
   - `/huntadmin setspawn resource` - 设置资源点

## 游戏流程

1. **准备阶段**
   - 管理员使用 `/huntadmin start` 开启游戏大厅
   - 玩家使用 `/hunt join` 加入游戏
   - 达到最小玩家数后开始30秒倒计时

2. **游戏开始**
   - 随机分配角色（杀手/求生者）
   - 传送到各自的出生点
   - 给予初始物品
   - 资源点开始生成物品

3. **游戏进行**
   - 杀手：寻找并击杀求生者
   - 求生者：躲避杀手、收集资源、生存
   - 每2分钟显示10秒杀手轮廓
   - 游戏时间倒计时

4. **游戏结束**
   - **求生者获胜条件**：
     - 生存到时间结束
     - 击杀所有杀手
   - **杀手获胜条件**：
     - 在时间结束前击杀所有求生者
   - 显示游戏统计信息
   - 5秒后自动重置游戏

### 编译与部署
1. 使用 ScriptIrc 编译器编译插件

3. 重启服务器或使用 `/reload` 命令

## 注意事项

1. **性能考虑**：
   - 轮廓显示使用 Bukkit 的 `setGlowing` 方法，对性能影响较小
   - 事件监听器已优化，避免不必要的计算

2. **兼容性**：
   - 支持 Minecraft 1.21.x
   - 需要 Bukkit/Spigot/Paper 服务端

3. **安全考虑**：
   - 游戏中的玩家无法使用指令（私聊除外）
   - 无法破坏或放置方块
   - 物品不会掉落

4. **数据保存**：
   - 使用 YAML 配置文件保存出生点
   - 玩家数据仅在内存中，重启后重置
